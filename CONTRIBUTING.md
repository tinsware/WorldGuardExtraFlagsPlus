# Contributing to WorldGuardExtraFlagsPlus

Thank you for considering contributing! This document outlines the project structure, patterns, and guidelines to help you submit clean, mergeable PRs.

---

## Project Structure

The project has two Maven modules:

```
WG/          — WorldGuard / WorldEdit integration (flag declarations, session handlers)
Spigot/      — Bukkit server module (config, listeners, plugin lifecycle)
```

- **Flag declarations** go in `WG/src/main/java/dev/tins/worldguardextraflagsplus/flags/Flags.java`
- **Custom flag types** go in `WG/src/main/java/dev/tins/worldguardextraflagsplus/flags/helpers/`
- **Session handlers** go in `WG/src/main/java/dev/tins/worldguardextraflagsplus/wg/handlers/`
- **Bukkit event listeners** go in `Spigot/src/main/java/dev/tins/worldguardextraflagsplus/listeners/`
- **Config** is managed via ConfigLib: `PluginConfig.java` + `Config.java`
- **Plugin lifecycle** (flag registration, listener registration) is in `WorldGuardExtraFlagsPlusPlugin.java`

## How to Add a New Flag

Follow this checklist in order:

### 1. Declare the flag in `Flags.java`

Pick the correct type:

| Type | When to use | Example |
|---|---|---|
| `StateFlag` | On/off per region (allow / deny) | `GODMODE`, `FLY`, `FROSTWALKER` |
| `BooleanFlag` | True/false per region | `KEEP_INVENTORY`, `DISABLE_COLLISION` |
| `StringFlag` | Free-form text value | `CHAT_PREFIX`, `CHAT_SUFFIX` |
| `SetFlag<...>` | List of values per region | `ALLOW_BLOCK_BREAK`, `DISABLE_COMPLETELY` |
| `DoubleFlag` | Decimal number | `WALK_SPEED`, `FLY_SPEED` |
| `IntegerFlag` | Whole number | `PLAYER_COUNT_LIMIT` |
| `LocationFlag` | World coordinate | `TELEPORT_ON_ENTRY`, `JOIN_LOCATION` |

Example:

```java
public final static StateFlag YOUR_FLAG = new StateFlag("your-flag", true);
```

The second argument is the default state (`true` = allow / `false` = deny).

### 2. (If needed) Create a custom flag type

If your flag needs special validation or serialization, create a new class in `flags/helpers/` extending the appropriate base type (e.g., `BlockableItemFlag`, `ThrowableItemFlag`, `MaterialFlag`).

### 3. Add a config toggle in `PluginConfig.java`

In the `AllFlagsControl` inner class, add a new `private boolean` field with an optional `@Comment`:

```java
@Comment({
    "your-flag flag",
    "Description of what it does.",
    "Usage: /rg flag your-flag deny",
    "Default: true"
})
private boolean yourFlag = true;
```

### 4. Add a switch case in `Config.java`

In the `isFlagEnabled()` method:

```java
case "your-flag": return flags.isYourFlag();
```

The method name is automatically generated by Lombok's `@Getter` from the field name (converting camelCase to `is`-prefixed).

### 5. Register the flag in `onLoad()`

In `WorldGuardExtraFlagsPlusPlugin.java`, inside the try-block:

```java
if (Config.isFlagEnabled("your-flag")) flagRegistry.register(Flags.YOUR_FLAG);
```

### 6. (If needed) Create a session handler

If your flag requires per-player session tracking (e.g., fly, godmode, effects), create a handler in `wg/handlers/` extending the appropriate base class and register it in `onEnable()`:

```java
if (Config.isFlagEnabled("your-flag"))
    this.sessionManager.registerHandler(YourFlagHandler.FACTORY, null);
```

### 7. (If needed) Create a Bukkit listener

If your flag responds to Bukkit events, create a listener class implementing `Listener`:

```java
package dev.tins.worldguardextraflagsplus.listeners;

import com.sk89q.worldedit.bukkit.BukkitAdapter;
import com.sk89q.worldguard.LocalPlayer;
import com.sk89q.worldguard.bukkit.WorldGuardPlugin;
import com.sk89q.worldguard.protection.flags.StateFlag.State;
import com.sk89q.worldguard.protection.regions.RegionContainer;
import com.sk89q.worldguard.session.SessionManager;
import dev.tins.worldguardextraflagsplus.flags.Flags;
import org.bukkit.entity.Player;
import org.bukkit.event.EventHandler;
import org.bukkit.event.Listener;
import org.bukkit.event.entity.EntityDamageEvent;

public class YourFlagListener implements Listener
{
    private final WorldGuardPlugin worldGuardPlugin;
    private final RegionContainer regionContainer;
    private final SessionManager sessionManager;

    public YourFlagListener(WorldGuardPlugin worldGuardPlugin,
                            RegionContainer regionContainer,
                            SessionManager sessionManager)
    {
        this.worldGuardPlugin = worldGuardPlugin;
        this.regionContainer = regionContainer;
        this.sessionManager = sessionManager;
    }

    @EventHandler(priority = EventPriority.NORMAL, ignoreCancelled = true)
    public void onEvent(EntityDamageEvent event)
    {
        if (!(event.getEntity() instanceof Player player))
            return;

        LocalPlayer localPlayer = this.worldGuardPlugin.wrapPlayer(player);

        if (this.sessionManager.hasBypass(localPlayer, localPlayer.getWorld()))
            return;

        State state = this.regionContainer.createQuery().queryState(
                BukkitAdapter.adapt(player.getLocation()),
                localPlayer,
                Flags.YOUR_FLAG);

        if (state == State.DENY)
            event.setCancelled(true);
    }
}
```

### 8. Register the listener in `onEnable()`

```java
if (Config.isFlagEnabled("your-flag"))
{
    this.getServer().getPluginManager().registerEvents(
            new YourFlagListener(
                    this.worldGuardPlugin, this.regionContainer, this.sessionManager), this);
}
```

Use the short class name — the wildcard import `dev.tins.worldguardextraflagsplus.listeners.*` is already at the top of the file.

### 9. Add metadata to `FlagDescriptions`

Every flag must have a human-readable entry in `Spigot/src/main/java/dev/tins/worldguardextraflagsplus/api/FlagDescriptions.java`.
Add a line in the static initialiser block:

```java
ALL.put("your-flag", new FlagMeta(
    "Short description of what the flag does.",
    "Accepted value types (e.g. allow/deny, true/false, material names)",
    "/rg flag <region> your-flag example-value"));
```

This metadata is used by the plugin's own commands and by external GUI plugins (e.g. WG-GUI) to display flag tooltips.

## Folia Compatibility

This plugin supports Folia. Follow these rules:

- **DO NOT** use `Bukkit.getScheduler()`, `BukkitRunnable`, or any deprecated Bukkit scheduling API
- Use `WorldGuardUtils.getScheduler()` for all scheduling needs
- Entity-specific code must run on the entity's owning thread (safe inside Bukkit event handlers)
- Do not perform cross-world chunk loading or world operations outside proper scheduling
- Use `runNextTick()` and `runAtLocation()` from the FoliaLib scheduler

---

## Code Style

- Indentation: tabs (not spaces)
- Opening brace on the same line for class/method signatures, next line for control flow
- `@Override` on separate line above method signature
- No trailing whitespace
- Use `final` fields where possible in listener/handler classes

---

## PR Guidelines

- **One commit per feature or fix.** Squash your changes before opening the PR.
- **Target the correct branch:**
  - `master` — stable features and fixes ready for release
  - `dev-point` — experimental or in-progress work
- **Commit messages:** Write descriptive commit messages explaining the *why*:
  ```
  feat: add lightning-damage region flag

  New StateFlag that allows lightning strikes to appear visually
  in a region while preventing damage to players. Useful for PvP
  arenas wanting the visual effect without environmental damage.

  Usage: /rg flag <region> lightning-damage deny
  ```
- **No personal attributions, watermarks, or trademarks** in new files. Clean code only.
- **Test that it compiles** before submitting:

  ```bash
  mvn clean package -DskipTests
  ```

- If the PR touches multiple unrelated features, split into separate PRs.

```bash
mvn clean package
```

The output jar will be in `Spigot/target/WorldGuardExtraFlagsPlus-{version}.jar`.
